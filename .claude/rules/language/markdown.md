# Markdown 規約

## 文字種・表記

### 基本ルール

- 数値: 半角を使用する（例: `123`、`2024年`）
- 記号: 半角が存在するものは半角を優先する（例: `!`、`?`、`:`、`;`、`/`）
- カタカナ: 全角を使用する
- アルファベット: 半角を使用する

### スペース

- 全角文字と半角英数字の間に半角スペースを入れる
  - 良い例: `これは TypeScript です`
  - 悪い例: `これはTypeScriptです`

### 括弧

- 括弧は半角 `()` を使用し、前後に半角スペースを入れる
  - 良い例: `詳細は設定ファイル (config.json) を参照`
  - 悪い例: `詳細は設定ファイル(config.json)を参照`

### 長音符

- カタカナ語末尾の長音符は省略しない
  - 良い例: `サーバー`、`ユーザー`、`コンピューター`
  - 悪い例: `サーバ`、`ユーザ`、`コンピュータ`

## 作図

- 図やダイアグラムを作成する場合は、必ず **mermaid 記法** を使用してください。
- ASCII アートや画像ファイルではなく、mermaid で表現可能な図は mermaid を優先してください。
- 主な用途:
  - フローチャート (`flowchart`)
  - シーケンス図 (`sequenceDiagram`)
  - クラス図 (`classDiagram`)
  - ER 図 (`erDiagram`)
  - 状態遷移図 (`stateDiagram-v2`)
  - ブロック図 (`block-beta`)
  - その他 mermaid がサポートする図
